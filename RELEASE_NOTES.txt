braai — Release Notes
=====================

All notable changes to this project are documented in this file.

Format is based on "Keep a Changelog" (https://keepachangelog.com), and this
project adheres to Semantic Versioning (https://semver.org): MAJOR.MINOR.PATCH.
  - MAJOR: incompatible / breaking changes
  - MINOR: new, backward-compatible functionality
  - PATCH: backward-compatible bug fixes

The newest release is listed first. Dates use ISO 8601 (YYYY-MM-DD).
Categories: Added, Changed, Deprecated, Removed, Fixed, Security.

[0.3.0] - 2026-07-07
---------------------

Added
  - fetch_url tool: the model can now fetch a URL and receive its text content
    alongside the HTTP status code. HTML pages are passed through a readability
    filter (main article body extracted, navigation/sidebars/footers stripped)
    and then converted to Markdown; other content types are returned as-is. The
    tool surfaces all errors (network, TLS, 4xx/5xx) in the result text so the
    LLM can decide how to respond — no retries are performed by the tool itself.
  - Readability pre-filter for HTML pages (codeberg.org/readeck/go-readability/v2).
    Strips navigation menus, sidebars, reference lists, footers, and other page
    chrome before Markdown conversion, dramatically reducing the token count
    returned to the model for content-heavy pages such as Wikipedia articles.
  - Post-extraction text cap (32 KiB). Applied to the extracted text after
    readability + Markdown conversion, independently of the raw body download
    cap (fetch_url_max_bytes). When the cap is hit a "[content truncated]" note
    is appended so the model knows the page continues beyond what was returned.
  - fetch_url_enabled config key (default false). Privacy gate: when false the
    tool definition is never sent to the model, so the model cannot make outbound
    HTTP requests at all. Set to true in braai.conf or via /config to opt in.
  - fetch_url_https_only config key (default true). Rejects plain http:// URLs
    and blocks HTTPS-to-HTTP downgrade redirects at the tool level. Set to false
    only for local or intranet HTTP targets.
  - fetch_url_max_bytes config key (default 2097152 / 2 MiB). Caps the raw
    response body size downloaded before text extraction.
  - fetch_url_timeout_seconds config key (default 30). HTTP request timeout for
    fetch_url calls. All four fetch_url_* keys are hot-applicable: /config
    changes take effect immediately without restarting braai.
  - Apache License 2.0. LICENSE file added; startup banner and --version output
    now display "Licensed under the Apache License 2.0". Copyright held by
    The braai Authors.

Security
  - fetch_url is disabled by default (fetch_url_enabled = false). Outbound HTTP
    requests require explicit opt-in, keeping the default behaviour fully
    offline and privacy-safe.
  - When fetch_url_https_only = true (the default), plain http:// URLs are
    rejected and any redirect that downgrades from HTTPS to HTTP is blocked at
    the tool level, preventing silent bypass of the TLS requirement.

[0.2.2] - 2026-07-06
---------------------

Changed
  - Startup banner now prints the full /help command listing immediately after
    the mascot, so available commands are visible without having to type /help.
    The help output is rendered by a new printHelp() helper that is shared with
    the /help slash-command, so the two are always in sync.

[0.2.1] - 2026-07-06
---------------------

Added
  - Mascot ASCII art displayed in the startup banner, side-by-side with the
    version, working directory, model, and hint line.
  - @path TAB completion now shows readline's native column-list when multiple
    matches exist. The completer returns only the untyped suffix of each
    candidate, so readline replaces only the partial basename and displays
    clean short names in the picker.
  - /tree [<glob>] command now accepts an optional glob pattern to filter
    which root-level entries are shown (e.g. /tree internal* or /tree internal/ag*).
    Patterns with a path separator navigate into the prefix directory and match
    only the final name component. TAB after /tree completes directory names,
    including sub-paths (e.g. /tree internal/ + TAB lists dirs inside internal/).
  - /config mode [dark|light] setting. Persisted to braai.conf; hot-applied
    immediately (no restart required). In light mode the >>> prompt switches from
    cyan to blue, keeping it legible on white/light terminal backgrounds.
    terminal.Blue() helper added to internal/terminal/style.go.

Changed
  - /help @<path> entry expanded from one line to four: explains what attachments
    are (file content inlined into the prompt), supported formats, multiple tokens
    per message, and space-escaping with Tab completion.
  - /config entry in /help now includes a usage hint: "(e.g. /config mode light)".
  - /tree entry in /help updated to /tree [<glob>]; the "hidden" sub-argument
    (include dotfiles) has been removed.
  - All error and warning messages in handleSlashCommand are now styled red
    (terminal.Red) consistently across every command: /forget-history, /model,
    /save, /copy, /export, /tree, /cache, /config, and the unknown-command
    fallback.
  - /tree glob output now prints a synthetic root label (e.g. "internal/") when
    a path-prefixed glob is used, so the tree is visually anchored.

Fixed
  - @path TAB completion double-typed the directory prefix (e.g. @RELEARELEASE_NOTES.txt).
    Root cause: the completer returned the full escaped filename as the suffix
    instead of only the characters after what the user had already typed.
  - /tree [TAB] appended a trailing "/" to the completion, causing the command to
    receive "img/" as the glob — which never matched because filepath.Match
    compares against bare entry names. Trailing slash stripped from completions
    and from the parsed glob pattern.
  - /tree [TAB] did not complete sub-paths (e.g. /tree internal/ + TAB listed
    root entries instead of entries inside internal/). Completer now splits the
    typed word into a directory prefix and basename, navigates into the prefix
    directory, and returns only the basename suffix.
  - /config mode did not take effect: applyConfigField fell through to switchModel
    for the "mode" key, which could return a network error — causing the rl.SetPrompt
    hot-apply to be skipped. Fixed by returning nil early from applyConfigField for
    "mode".
  - /tree internal/ag* failed with "no entries matching": the old code passed the
    full pattern to filepath.Match against root entry names, so patterns containing
    a path separator never matched. Fixed by splitting the pattern into globDir and
    nameGlob and reading entries from the resolved subdirectory.

[0.2.0] - 2026-07-06
---------------------

Added
  - @path file attachments in interactive chat. Type @path/to/file anywhere in a
    prompt to inject the file's full content inline before it reaches the model.
    Works for all formats the read tool supports: plain text, code, PDF, Word,
    Excel, PowerPoint, HTML, RTF. Paths with spaces are escaped with a backslash
    (standard shell convention: @notes/Meeting\ Summary.md), which Tab completion
    produces automatically. If the path cannot be resolved or read, the turn is
    hard-stopped with a clear error.
  - Tab completion for both @path tokens and /commands. Pressing Tab after @
    (or @partial/path) completes file and directory names from the working
    directory; directories get a trailing / so you can keep drilling down; spaces
    in names are escaped automatically. Pressing Tab after / (or a partial
    command like /fo) completes slash-commands (/forget-history, /config, etc.).
  - /config command (interactive chat). View and change settings without editing
    braai.conf by hand:
      /config                    — list all settings with current values, grouped
                                   by section; marks hot-applicable vs restart-
                                   required keys.
      /config <key>              — show a single key's current value and description.
      /config <key> <value>      — change a setting; hot-applicable keys (all
                                   tool limits, max_tool_calls, num_ctx,
                                   keep_alive) take effect immediately without
                                   restarting; other keys save to disk and note
                                   that a restart is required.
  - /tree [hidden] command. Renders the working directory as an ASCII tree
    (├──, └──, │) directly in the REPL. Standard skip list (.git, node_modules,
    vendor, .idea) is always excluded. Pass "hidden" to also show dotfiles.
  - /digest command and --summarize flag. Submit a fixed structured-overview
    prompt that walks the tree and reads key files to produce a project summary.
    /digest works inside interactive chat (conversation continues afterwards);
    --summarize runs non-interactively and exits.
  - Answer citations (Sources: footer). After each answer that used file-reading
    tools (read, search, get_chunk, stat_file), a dim "Sources:" line lists the
    files and line numbers the model drew from, each enclosed in [brackets] so
    filenames with spaces stay clearly delimited:
      Sources: [README.md] [notes/Meeting\ Summary.md:12] [internal/agent/agent.go:143]
    Suppressed in --output json mode and when piped/redirected.
  - /export json <file> command. Saves the conversation as a JSON array of
    {role, content} objects (user and assistant turns only), skipping the system
    message and tool plumbing.
  - /copy [last] command. /copy copies the full conversation transcript to the
    clipboard (unchanged); /copy last copies only the most recent assistant
    answer — useful for pasting a single response into another tool.

Security
  - braai.conf now written 0600 (owner-only) instead of 0644. The config file
    contains the Ollama host URL and model name, and is now consistent with the
    0700 directory and 0600 permissions applied to the cache key, chat history,
    and all other sensitive files braai manages.
  - Embedding model download pinned to an immutable Hugging Face commit SHA
    (6fc8051fab2a1e0ee76689cf08c853792ac285e7) instead of the mutable "main"
    branch. Prevents a silent model swap if the upstream repo is updated.

Fixed
  - search() files_inspected counter now counts every file that passes the
    size/readability check, not only files that passed the binary-content sniff.
    The JSON field is renamed from files_scanned to files_inspected to reflect
    the corrected semantics.
  - list_dir name_contains filter no longer hides parent directory entries from
    results. Directories are now always emitted so that matching files have
    visible parent paths in the listing (previously they appeared as orphaned
    entries without a parent directory row).
  - Signal-handler goroutine in interactive chat is now always unblocked via
    defer close(turnDone), preventing a theoretical goroutine leak on panic.
  - Unconditional trailing newline after each answer: finish() in the stream
    printer now always emits a newline, so the >>> prompt never appears on the
    same line as residual output when a model returns an empty answer.

Changed
  - Interactive chat spinner now shows a text label: "Thinking…" on the first
    turn, "Reasoning…" while the model processes a tool result. The label is
    updated dynamically for each round-trip.
  - Each tool call is now printed as a compact one-liner (⚙ tool_name) even in
    non-verbose mode, so users can see what the agent is doing in real time
    without enabling --verbose.
  - Ctrl-C during a stream now prints "^C  (interrupted)" in dim text instead of
    a bare newline, confirming the cancellation was received.
  - Context-window warning is now styled in yellow for better visibility.
  - Welcome banner now uses colour: bold version, cyan working directory, green
    model name, dim hint line. Degrades cleanly when colour is disabled.
  - Interactive >>> prompt is cyan; turns red after an agent error and resets on
    the next successful turn.
  - /help command widths are now computed dynamically from the longest command
    name, so alignment never breaks if a command name is added or changed.
  - /clear now mirrors the compact banner (model + working directory) after
    clearing, so users know which model is still active.
  - /model list now marks the active model with a bold green ✓ and dims the
    others, replacing the bare "* " marker.
  - Embedding model download now shows a "Downloading embedding model…" spinner
    on stderr during the first-run download, replacing the previous silent hang.
  - Error messages in interactive chat no longer carry a redundant "error: "
    prefix; the red colour already signals failure.
  - "Thinking..." / "...done thinking." labels extracted to named constants in
    stream.go (labelThinking, labelDoneThinking).

Removed
  - Dead config.Path() redirect (deprecated alias for ConfPath()) deleted.
  - Legacy string-keyed dispatcher cases for old tool names (read_file,
    read_files, read_document, search_name, search_content, search_document,
    search_semantic) removed from Registry.Call(). These were unreachable from
    normal use because the model is never given schemas with those names;
    internal helpers remain and are called through the read/search facades.
  - All dead *Definition() schema-builder functions (readFileDefinition,
    readFilesDefinition, readDocumentDefinition, searchNameDefinition,
    searchContentDefinition, searchDocumentDefinition, searchSemanticDefinition)
    removed. The model-visible tool set is unchanged.
  - errStopWalk sentinel moved from its own walk.go file into search_content.go
    where it is used; walk.go deleted.

Performance
  - config.Dir(), CacheDir(), and ModelsDir() now skip the os.Chmod syscall when
    directory permissions are already 0700, avoiding redundant syscalls on every
    invocation.
  - dirEntryInfo struct no longer carries an unexported modTime sort-key field;
    sorting uses a separate (entry, modTime) pair slice that is stripped before
    JSON marshalling.
  - newChatSession() now takes a chatSessionOptions struct instead of 14
    positional arguments, preventing silent argument mis-ordering bugs.


[0.1.3] - 2026-07-05
---------------------

Security
  - Pin embedding model download to revision constant: model.safetensors,
    tokenizer.json are now pulled from a pinned hfRevision constant (currently
    "main") rather than hard-coded "/resolve/main/". Prepare for supply-chain
    hardening by pinning to an immutable commit SHA.
  - Consolidate crypto code: migrate cache.go to use shared crypt.LoadOrCreateKey
    and crypt.WriteFileSecure instead of byte-identical duplicates. Prevents
    AES-256-GCM key handling from drifting between cache and history.
  - Harden symlink validation for non-existent paths: non-existent target paths
    now validate that their parent directory isn't a symlink escaping the root.
    Closes the edge case where a symlink directory inside root pointing outside,
    combined with a non-existent child, could bypass containment.

Fixed
  - Lenient tool-argument coercion: tool arguments now accept numeric strings
    (e.g. "3" instead of 3.0) and boolean strings (e.g. "true", "yes", "1")
    from smaller local models that emit JSON differently. Silent degradation
    to defaults is replaced with robust coercion across all argument sites
    (max_tokens, threshold, top_k, clean, etc.).
  - Inline code styling now spans stream chunks: backtick spans split across
    streamed deltas are buffered and merged so inline code at chunk boundaries
    is styled correctly (bold applied to both open and close backticks).
  - Config read errors are now reported: if braai.conf is unreadable (e.g.
    permission denied), a one-line warning is printed to stderr before
    degrading gracefully to defaults.


[0.1.2] - 2026-07-05
---------------------

Fixed
  - Cap batch-read document text (major): read() with many large PDFs no longer
    floods the model context. Each extracted document in a batch is truncated to
    128 KiB (configurable: max_document_bytes in braai.conf) with a note directing
    users to read the file alone for the full chunked manifest.
  - Empty path in semantic search (minor): search(semantic=true, path="") now
    correctly routes to whole-tree semantic search instead of erroring.

Added
  - Per-turn tool-call deduplication (efficiency): identical tool calls within a
    single turn are cached and re-executed results reused, preventing models from
    re-verifying/re-reading the same data. Verbose mode shows "[cached, skipped
    re-execution]" in yellow for deduplicated calls.
  - Color verbose output (feature): tool calls now print in yellow; errors in red.
    Improves visual distinction when debugging with --verbose.
  - Configurable Ollama runtime options:
    - `num_ctx`: set context window in tokens (e.g. 16384) to prevent truncation
      of long tool results. Default: 0 (Ollama default).
    - `keep_alive`: keep the model loaded between calls (e.g. "30m" or "-1").
      Default: "" (Ollama default). Avoids reload latency.
  - Comprehensive ~/.braai/braai.conf template: first run now generates a
    fully-annotated config file with all tunable tool limits and their defaults,
    organized into logical sections (Core, Ollama runtime, Chat recall, Semantic
    cache, Tool limits). Users can easily find and edit any setting.
  - All tool limits now persisted in braai.conf with sensible defaults:
    - max_read_bytes: -1 (unlimited for single files)
    - max_search_file_bytes: 2 MiB
    - max_search_results: 200 (exact search)
    - max_name_results: 500 (list_dir name_contains)
    - max_batch_files: 20 (read batch size)
    - max_image_bytes: 10 MiB
    - max_semantic_files: 200 (whole-tree search scope)
    - max_semantic_results: 10 (passages returned)
    - max_embed_chars: 8,000 (embedding token limit)
    - max_document_bytes: 128 KiB (batch read truncation)

Changed
  - Consolidated read tools: `read_file`, `read_files`, and `read_document` are
    now a single `read` tool that intelligently dispatches based on arguments
    and file type. Single text files support line ranges; documents (PDF/Office/
    HTML/RTF) are extracted automatically; `paths` array reads multiple files in
    one call. The get_chunk workflow for large documents is preserved. Reduces
    tool-choice ambiguity for the model.
  - Consolidated search tools: `search_content`, `search_semantic`, and
    `search_document` are now a single `search` tool with a `semantic` flag and
    optional `path` parameter. Exact (fast) substring search is the default
    (semantic=false); set semantic=true for meaning-based search. Scope to a
    single document with the path parameter. Maintains backward-compatible
    behavior and all existing output formats.
  - Folded `search_name` into `list_dir`: added a `name_contains` parameter
    (case-insensitive substring filter) to list_dir. Recursive name search is
    now just list_dir(path, depth=100, name_contains="pattern").
  - Updated system prompt to document the consolidated 6-tool set (was 13 tools).

Removed
  - `search_name`, `read_file`, `read_files`, `read_document`, `search_content`,
    `search_semantic`, `search_document` are no longer advertised to the model.
    Their functionality is unified in `read` and `search`, or integrated into
    `list_dir`. (Internal routes preserved for backward compatibility.)

Notes
  - The unified tools + dedup + context capping + config tuning reduce both
    tool-selection ambiguity AND context overflow, making braai significantly
    more reliable on long conversations with repeated queries.
  - All changes are backward compatible: old internal routes and methods remain
    unchanged. Only the public Definitions() list reflects the merges.


[0.1.1] - 2026-07-05
--------------------

Added
  - New `ollama_timeout` setting (default: 300 seconds / 5 minutes) to configure
    HTTP request timeout for Ollama calls. Useful for slow networks or models
    that need more time to process requests. Configurable in ~/.braai/braai.conf.

Changed
  - Tool output now uses compact JSON (single-line, no indentation) to save
    tokens. Users can pipe to `jq` for pretty-printing: `braai --output json
    "..." | jq .`
  - Consolidated file enumeration: removed find_all_files tool. Use list_dir
    with a large depth (e.g. depth=100) to list entire directory trees
    recursively. This reduces tool-choice ambiguity for the model.

Removed
  - find_all_files tool. Equivalent functionality now available via
    list_dir(path, depth=100) for recursive enumeration of entire trees.


[0.1.0] - 2026-07-04
--------------------

Added
  - Custom prompt-template commands (/cmd). Users can now create reusable prompt
    templates as Markdown files in ~/.braai/commands/ (global) or <working-dir>/
    .braai/commands/ (per-project, overrides global). Templates support variable
    substitution: $ARGS (all args), $1-$9 (positional), $SELECTION (last answer),
    and $$ (literal $). Invoked as "/cmd <name> [args...]"; "/cmd" alone lists
    available commands. Frontmatter (description, args) is optional. Templates
    are pure prompts—they expand and submit as normal turns, with no new
    security surface.


[0.0.7] - 2026-07-04
--------------------
Changed
  - Configuration moved from ~/.braai/settings.json (JSON) to
    ~/.braai/braai.conf (key=value). Files are now human-editable and support
    comments; your comments are preserved across restarts.
  - Config saves are merge-based: only known keys are updated, while comments
    and unknown keys are left untouched.
  - Config writes are now atomic, preventing file truncation if a save is
    interrupted.

Added
  - New history_limit setting (default 100) controlling chat recall history size.
  - Chat recall history now persists across sessions for up/down arrow recall.
  - /forget-history command to clear the chat recall history.

Security
  - Chat recall history is now encrypted at rest using AES-256-GCM, stored at
    ~/.braai/chat_history with no plaintext prompts on disk. Uses a machine-local
    key at ~/.braai/cache.key (shared with the semantic cache).
  - Removed debug output that printed decrypted history in plaintext to stderr.

Fixed
  - Config comments were being destroyed on every startup.
  - History file path used a non-portable "/dev/null" setting on Windows
    (now uses an empty string).
  - History file path used string concatenation instead of filepath.Join,
    producing wrong separators on Windows.

Notes
  - Semantic cache is unchanged: still AES-256-GCM encrypted at rest,
    persistent, compressed, with LRU eviction, and using in-process static
    embeddings (no Ollama required for embeddings).

Upgrade
  - Recreate your configuration in ~/.braai/braai.conf using the new key=value
    format; the previous ~/.braai/settings.json is no longer read.
  - Any existing plaintext ~/.braai/chat_history is discarded on first launch
    and replaced with an encrypted store.


[0.0.1] - 2026-07-01
--------------------
Added
  - Initial release: read-only local AI agent CLI over a working directory,
    powered by a local Ollama chat model.
  - Read-only toolset: list_dir, read_file, read_files, search_name,
    search_content, search_semantic, read_document, search_document, get_chunk,
    find_all_files, stat_file, and read_image (vision-capable models only).
  - Semantic search with in-process static embeddings and a persistent,
    compressed, AES-256-GCM-encrypted on-disk cache with LRU eviction.
  - Interactive chat with slash commands (/help, /clear, /copy, /tools,
    /tools full, /cache, /model, /save, /forget-history, /bye) and streamed
    reasoning + answers.
